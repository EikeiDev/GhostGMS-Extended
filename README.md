# GhostGMS Extended

Hey there! This is **GhostGMS Extended**, an advanced fork of the original [GhostGMS by kaushikieeee](https://github.com/kaushikieeee/GhostGMS). 

While we’ve kept the core spirit of the original module, this version has been heavily rebuilt from the ground up to offer a more professional, stable, and visually stunning experience.

The goal was simple: **Maximum battery life, absolute privacy, and zero lag.** No placeholders, no fake buttons—just pure optimization.

---

## 🚀 What's inside this release?

We didn't just tweak a few lines; we overhauled the entire experience. Here’s what makes this version special:

### 💎 The "Aura" WebUI
Forget those old, static templates. We’ve built a fully interactive Control Center.
*   **Live Sync**: The UI reflects your real system settings. If a category is blocked in the config, you'll see it instantly.
*   **Interactive Categories**: Use the "Service Master Switch" to toggle entire groups of GMS services with a single tap.
*   **Premium Design**: Dark mode, smooth transitions, and a layout that actually feels like part of a high-end OS.

### 🧠 The Smart Engine (Veloxine)
The heart of the module is the **Veloxine Service Engine**.
*   **Granular Blocking**: We categorized GMS services into logical groups (ADS, TRACKING, SYNC, etc.) so you can decide exactly what to kill and what to keep.
*   **Dual-Path Persistence**: Your settings are saved in two places. Even if you update or reinstall the module, your choices stay with you.
*   **KSU Next Support**: We implemented a unique "Hex Bypass" to solve configuration loading issues on KernelSU Next. It's rock-solid.

### 🔋 Battery & Performance
*   **Logging Neutralization**: We don't just "mute" logs; we can shut down the system logger to save CPU cycles.
*   **Prop Optimizations**: Fine-tuned system properties to keep Dalvik and GMS from acting up in the background.
*   **Minimalist Code**: No comments, no dead weight. Just clean, high-performance shell scripts.

---

## 🛠 How to use it?

1.  **Install**: Flash the module in KernelSU (Next) or Magisk.
2.  **Configure**: Use your volume keys during installation to set your initial preferences.
3.  **Manage**: After reboot, open the **WebUI** in your manager to fine-tune your settings.
4.  **Enjoy**: Save changes, and the module will handle the rest.
